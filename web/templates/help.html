What is this tool?

The Stakeholder Mapping Tool helps you track and manage the people involved in your programs. You can map who matters, assign responsibilities, track engagement, spot risks, and visualize relationships.

Influence & Support Levels

Influence is how much power a stakeholder has over the project's success:

  • high — Can make or break the project. Executive sponsors, budget holders.
  • medium — Significant input. Team leads, key SMEs.
  • low — Limited direct influence. Keep informed.

Support is their stance toward the project:

  • champion — Actively advocates for the project.
  • supporter — Generally positive and cooperative.
  • neutral — No strong opinion yet. Could go either way.
  • resistant — Opposes or blocks the project. Needs engagement.

Blockers are stakeholders with high influence and resistant or neutral support. Focus your engagement here.

Exporting & Sharing Data

Click the Export button in the top-right corner to download a full JSON backup of your data.

To share with a teammate, use the terminal:

  • make export — saves everything to stakeholder-export.json
  • Send that file to your teammate
  • They run make import FILE=stakeholder-export.json to load it

Other export formats available while the tool is running:

  • CSV (for Excel): /api/v1/export/stakeholders.csv
  • JSON (stakeholders only): /api/v1/export/stakeholders.json

AI Assistant (Optional)

The tool includes an AI chat assistant that can generate meeting briefs, draft emails, and analyze blockers. It requires Ollama, a free tool that runs AI models on your computer.

Setup:

  1. Download and install Ollama from ollama.ai
  2. Open a separate terminal and run: ollama serve
  3. Download a model: ollama pull llama3.2
  4. Restart this tool — the AI chat button will appear in the bottom-right corner

If Ollama is not running, the chat will show a friendly message. Nothing breaks.

How Data is Stored

Your data is stored in a local SQLite database file called stakeholder.db in the project folder. No external database server is needed — everything runs on your computer.

The database is created automatically the first time you start the tool. To start completely fresh, run make reset in the terminal.
